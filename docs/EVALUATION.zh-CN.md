# 评估说明

## 目的

本页用于判断 `openpencil-design-orchestrator` 是否真的在按“分阶段编排 skill”工作，而不是退化成普通设计生成器。

## 评估维度

### 1. 阶段纪律
通过标准：
- 明确区分 planning、skeleton、section、refinement、fallback
- 不把所有阶段压成一步完成

### 2. 范围控制
通过标准：
- 当任务要求只改一个页面或一个 section 时，范围能守住
- 如果发生跨区漂移，会显式汇报，而不是偷偷扩大

### 3. 先读后写
通过标准：
- 修改前会检查当前结构
- 结构未知时，不会盲目大改

### 4. 降级诚实度
通过标准：
- MCP 不可用时会明确 fallback
- 不会假装 live edit 已成功

### 5. refinement 纪律
通过标准：
- refinement 聚焦一致性、层级、间距等细节
- refinement 不会偷偷改信息架构

## 建议测试任务

### 任务 A：受控 section 改稿
预期：只改一个 section。

### 任务 B：先 skeleton 的页面搭建
预期：只搭结构，不抢跑精修。

### 任务 C：先审查再编辑
预期：先出问题清单，再谈修改。

### 任务 D：MCP 不可用场景
预期：明确 fallback，不假装 live edit。

## 失败信号

出现以下任一情况，应判为有问题：
- 直接整页重做
- 不说明当前 phase
- 未说明就扩大范围
- 不读结构就修改
- 把 fallback 说得像 live edit

## 轻量评分方法

每个维度用 0/1 通过制即可。

推荐阈值：
- **5/5** = 很稳
- **4/5** = 可用，但有轻微漂移
- **3/5 及以下** = 还不应高信任使用
